# 失落的方舟 - 方舟棋盘战斗力优化器

一个玩家自制的工具，用于优化失落的方舟中的方舟棋盘护石配置，以最大化战斗力。

## 关于本分支

这个分支是在原项目基础上的增强版本，主要改进包括：

### ✨ 主要改进

1. **新增服务器支持**：
   - 🇨🇳 **国服支持**（简体中文）
   - 🇷🇺 **俄服汉化支持**（中文界面的俄服）

2. **截图模式（新增）**：
   - 新增**单帧截图识别**模式，替代旧的流式录制
   - 点击「📸 截图识别」按钮即可逐帧捕获当前画面
   - 无需持续屏幕共享，操作更轻量、更稳定
   - 支持多次截图累积识别，内置 contentKey 精确去重

3. **精确去重系统（重构）**：
   - 采用 **contentKey 精确匹配**策略（9 字段：护石名 | 属性 | 意志力 | 点数 | 品质 | 选项1类型+等级 | 选项2类型+等级）
   - 相同内容直接覆盖更新，不同物理护石不会误判为重复
   - 解决了滚动截取时邻居哈希误判导致数量偏多的问题（如实际 48 个被误计为 54 个）

4. **完整中文本地化**：
   - 所有界面文本、说明、FAQ 都已翻译成中文
   - 游戏环境选择器支持所有服务器的中文显示

### ❌ 已放弃的功能

| 功能 | 状态 | 原因 |
|------|------|------|
| 长截图上传（PixPin 等） | **已移除** | 长截图分辨率与游戏窗口差异过大，anchor（"宝石"定位文字）模板匹配频繁失败或超时。OpenCV 在非标准分辨率的超大图像上模板匹配性能不足 |
| 旧版流式模式按钮 | **已隐藏** | 已被截图模式替代，函数保留以兼容内部逻辑 |

## 功能

1. **护石识别**：通过屏幕共享自动识别游戏中的护石
2. **截图识别**：单帧捕获 + 多次累积，轻量稳定
3. **战斗力优化**：智能计算最佳护石配置
4. **多服务器支持**：
   - 🇰🇷 韩服（韩语）
   - 🇺🇸 美服/国际服（英语）
   - 🇷🇺 俄服（俄语）
   - 🇷🇺 俄服汉化（中文界面）
   - 🇨🇳 国服（简体中文）
5. **配置管理**：保存和管理多个角色配置
6. **结果预览**：查看优化后的战斗力提升和护石配置

## 使用方法

### 1. 准备工作
- 打开失落的方舟游戏
- 进入角色的方舟棋盘界面
- 切换到未使用的预设以卸下所有护石

### 2. 选择服务器环境
在开始屏幕共享前，先选择正确的游戏环境：
- **国服玩家**：选择"国服（简体中文）"
- **俄服汉化玩家**：选择"俄服汉化"
- **其他服务器**：选择对应的服务器环境

### 3. 开始使用（推荐：截图模式）

1. 在浏览器中打开此工具
2. 点击 [🖥️ 开始屏幕共享] 按钮，选择失落的方舟游戏窗口
3. 切换到游戏中的**护石列表页面**（需看到"宝石"/"秩序"等界面元素）
4. 点击 [📸 截图识别] 按钮，工具会自动捕获当前帧并识别其中所有可见护石
5. 向下滚动游戏中的护石列表，再次点击 [📸 截图识别] 追加更多护石
6. 工具会自动去重，重复的护石会被覆盖而非重复添加
7. 确认所有护石都已识别后，点击 [✅ 应用到当前配置]
8. 进入优化设置页面，调整核心最小点数
9. 点击 [执行优化] 按钮，等待计算完成
10. 查看优化结果和建议

> **提示**：每点击一次截图识别会累加计数（如 `截图识别(3)`），方便确认已截取次数。

## 常见问题

### Q: 屏幕共享失败或被拒绝
A: 请使用 Chrome 或 Edge 浏览器，并确保已授予屏幕共享权限。

### Q: 护石没有被识别
A: 请检查以下内容：
- 游戏分辨率设置为 1920x1080 (16:9)
- 游戏窗口处于窗口模式
- 没有使用强制 21:9 宽高比
- 护石列表完全可见，没有被其他 UI 遮挡
- **确认已选择正确的游戏环境**（国服/韩服/美服等）
- **确认当前页面显示的是"宝石"/"秩序"/"混沌"等护石界面**（anchor 定位依赖这些文字）

### Q: 护石被重复识别了
A: 这个问题已经在本分支中修复！
- 使用 contentKey 精确匹配（9 字段），只有完全相同的护石才会去重
- 多个属性相同但物理位置不同的护石不会被误删

### Q: 截图后数量不对
A: 
- 确保每次截图时护石列表已完全加载完毕
- 如果某行护石部分被遮挡，可能导致该行识别缺失
- 可以多截几次，去重机制会自动合并

### Q: 优化结果不理想
A: 尝试调整核心最小点数设置，或确保所有护石都已正确识别。

### Q: 为什么不能上传长截图？
A: 长截图（如 PixPin 滚动截图）的分辨率通常远高于 FHD（例如 435×2951），且宽高比异常。这导致：
- OpenCV 模板匹配在超大图像上极慢（容易超时）
- anchor 定位文字在缩放后可能因分辨率过低而无法匹配
- 目前没有可靠解决方案，建议使用**截图模式**逐帧捕获

## 技术栈

- **求解器**: 带有上界剪枝的自定义回溯算法 (TypeScript)
- **前端**: Svelte 5 (基于组件的 UI，本地状态持久化)
- **图像处理**: OpenCV.js (模板匹配，Web Worker)
- **部署**: GitHub Pages (纯客户端), 腾讯云 CloudBase (静态网站托管)

## 如何运行

### 安装

使用 pnpm 安装依赖：

```bash
pnpm install
```

### 开发

启动开发服务器：

```bash
pnpm dev
```

### 构建

构建生产版本：

```bash
pnpm build
```

### 生成 OpenCV 精灵图

OpenCV 模板位于 `opencv-templates/` 目录下。运行以下命令将它们合并为一个精灵图：

```bash
pnpm run generate:sprite
```

生成的精灵图坐标将保存在 `lib/opencv-template-coords/` 目录中：

```typescript
// THIS FILE IS AUTO-GENERATED. DO NOT MODIFY ITSELF
export const zhCnCoords = {
  ...
  'anchor.png': { x: 0, y: 0, w: 148, h: 36 },
  ...
}
```
